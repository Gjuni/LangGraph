incident_organizer Tool의 역할을 정의한다.

역할: 사건사고 정리 에이전트(organize_node)가 사용하는 Tool이다.
    - web_scraper가 여러 사이트에서 검색해온 기사 목록 중 실제 날짜가 확인된 기사만 골라
      LLM(OpenAI)으로 5줄 내외 요약을 생성하고, 날짜별(최신순)로 그룹화한다.
    - 특정 날짜(예: 오늘)로 한정하지 않는다 — 검색어와 관련된 기사라면 날짜에 상관없이 포함하되, 최신 날짜가 먼저 오도록 정렬한다.

절대 규칙 (main.md 기준):
    - 서버(기사 본문)에서 실제로 확인되지 않은 날짜는 정리 대상에서 제외한다. 날짜를 추정/생성하지 않는다.
      → `_has_verified_date`로 LLM 호출 전에 코드 레벨에서 필터링한다.
    - title과 summary가 모두 채워지지 않은 기사는 결과에 포함하지 않는다.
      → 시스템 프롬프트로 LLM에 지시하고, 응답 파싱 후 코드 레벨에서 2차 검증한다.
    - 각 기사의 summary는 본문 내용을 근거로 총 5줄 내외로 작성한다(한 줄 요약이 아님).

입력/출력:
    - 함수: `organize_incidents(articles: list[dict]) -> dict`
    - 반환 형식: `{"YYYY-MM-DD": [{"date", "title", "link", "summary"}, ...], ...}` (여러 날짜 키 가능)
    - `report_writer`가 날짜 키를 내림차순 정렬하므로 최신 날짜가 보고서 상단에 온다.
    - 확인된 날짜의 기사가 하나도 없으면 빈 dict(`{}`)를 반환한다.

수정 시 주의:
    - 프롬프트(`_SYSTEM`)만 수정하면 요약 분량/톤을 바꿀 수 있다.
    - 날짜 검증(`_has_verified_date`)과 title/summary/link 검증은 LLM 응답을 신뢰하지 않는 방어 코드이므로 임의로 제거하지 않는다.
    - 이 Tool은 스크랩이나 파일 저장을 직접 수행하지 않는다.
